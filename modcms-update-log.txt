版本 1.4.2 (2017-6-21 10:48)
1. 将 ModPHP 内核更新为 2.2.0，并根据其调整一些设置。
2. 建议在模板中使用 do_hooks() 函数执行两个 API所绑定的回调函数（无参数）：
   mod.template.load.header --- 公共头部文件中(或后)调用
   mod.template.load.footer --- 公共尾部文件中(或前)调用
   例如:
   <include data="header.html"/>
   {do_hooks('mod.template.load.header')}
   <!-- 主体内容 -->
   {do_hooks('mod.template.load.footer')}
   <include data="footer.html"/>
   这两个 API 将作为日后编写前台插件时统一使用的 API，插件可以在这两个 API 所绑定的
   回调函数中，直接输出内容到页面中，以此来更改网站的外观和功能。

版本 1.2.0 (2017-2-20 22:00)【重要】
根据 ModPHP 新增加的特性，同时针对 NIGINX 服务器的特点，这一版本做出了众多改变；
旧版本可以升级到该版本，但可能会出现一些错误，主要的更新如下：
1. 后台全面增加多国语言支持，并支持自动语言切换(如果存在该语言包，不存在则默认英文)。
2. admin、template 和 plugins 目录全部转移到 app/ 文件夹下，统一使用 ModPHP 内置的模板编译器。
3. admin_dir() 和 plugin_dir() 函数将返回新的目录，而 admin_url() 和 plugin_dir_url() 则保持不变。
4. Apache 服务器上可一键启动 WebSocket 模式。
5. 新安装时自动设置语言(如果存在该语言包，不存在则默认英文)和时区。
6. 后台页面用户体验提升。
7. 修复了一些其他的 BUG。
8. 将 ModPHP 版本升级为 1.7.0，将内置模板 simplet 版本更新为 1.0.5(多语言支持)。
9. 从旧版本升级到该版本，ModCMS 会将原来的 admin、plugins 和 template 目录改名加上 .old 后缀。
   原来的插件和模板会自动复制到新目录下，但是不会复制 admin 目录下的文件。
   如果原来的 admin 目录下有自己创建的文件，你需要手动将其复制到新目录中。
   另外，如果你在原来的模板或者差插件中使用了 admin_dir()、admin_url()、plugin_dir() 和 
   plugin_dir_url() 函数，请注意这可能引发的错误。

版本 1.0.0 (2017-1-17 20:00)
这是 ModCMS 的第一个发行版本，主要包括了下面这些功能：
1. 用户管理：包括注册、更新、删除和找回密码等
2. 分类目录管理：包括创建、编辑、删除等
3. 文章管理：包括新建、编辑、删除等
4. 评论管理：包括查看、编辑、删除和回复等
5. 文件管理：包括查看、编辑和删除等
6. 模板和插件管理：包括上传、更换、删除和编辑等
7. 友情链接和黑名单管理：包含添加、编辑和删除等
8. 设置管理：包含完整的设置界面以及在线更新功能
